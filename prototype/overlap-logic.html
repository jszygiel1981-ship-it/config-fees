Why the distinction matters
Modified (auto-resolved) — The conflicting instance is already active or has no end date. Shifting its end date is safe because any timecards tied to it have already been processed up to today. TRIO can confidently close it out without disrupting historical data.

Blocked (manual required) — The conflicting instance is a future one — it was intentionally scheduled but hasn't started yet. TRIO won't assume it's safe to modify because the intent behind that future instance isn't known. You may have scheduled it for a reason. The decision to change it should be yours.
Created — new instance, no conflicts
Modified — existing instance end date auto-adjusted
Blocked — future instance conflict, manual resolution required
